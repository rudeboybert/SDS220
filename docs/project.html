Project

Everything in this course builds up to the group project, where there is only one learning goal: Engage in the data/science research pipeline in as faithful a manner as possible while maintaining a level suitable for novices.

1. Project Data

Form groups

All groups members are expected to contribute and you will all be held accountable for your contributions in peer evaluations.

  1. Form groups of 2-3 students. If you need a group to join, please Slack your instructor.
  2. Designate a group leader who will do a little administrative work.
  3. Pick a group name.
  4. The group leader will then create two Slack direct messages (DM):
    • One with all your group members + your instructor + your lab instructor. You’ll ask group-specific questions here.
    • One with only your group members. Send all private messages amongst yourselves here.
  5. Complete the project groups Google Form posted on Moodle.

Find data

This is likely the most time-consuming portion of the entire project, so we recommend you start early.

  1. Read moderndive for important background knowledge and tools for this project phase:
    • Chapter 1.4.4 on identification vs measurement variables.
    • Chapter 4.1 on importing spreadsheet data into R.
  2. Read the example data proposal.
  3. Download the following project_data.Rmd template R Markdown file.
  4. Find a dataset that matches the requirements spelled out in the above project_data.Rmd template R Markdown file. Possible sources include data from your own projects, Google Dataset Search, data.world, and Kaggle. Consult the Spinelli Center Data Counselor for help (see syllabus for contact info).
  5. The group leader only will submit the following files on Moodle on behalf of the whole group:
    1. The project_data.Rmd R Markdown file
    2. The project_data.html HTML report file
    3. Only if your data is confidential or has private information, submit your Excel or CSV file as well. Otherwise your should publish your data on Google Sheets.

Notes:

  • When you have questions:
    • Please ask questions that you think the entire class would like to know the answer to in the project channel in Slack.
    • Ask more individual questions in the group DM that includes all group members + your instructor + your lab instructor.
  • Only minimal data wrangling using the dplyr package is expected at this time; you will be doing more for the “project proposal” phase after you’re experienced. That being said, feel free to experiment!
  • Don’t forget to use the Spinelli 7-9pm tutoring hours (see syllabus for time/place).

3. Project Submission

Action items

  1. Read the following sections of the example project resubmission:
    • Section 1: Introduction
    • Section 2: Exploratory data analysis
    • Section 3 subsections 3.1, 3.2, and 3.3: Multiple linear regression: Methods, Model Results, Interpreting the regression table.
  2. Download the following project_submission.Rmd R Markdown file
    • Complete the following sections:
      • Section 1: Introduction
      • Section 2: Exploratory data analysis
      • Section 3 subsections 3.1, 3.2, and 3.3: Multiple linear regression: Methods, Model Results, Interpreting the regression table.
    • Do not complete the following sections (you’ll be doing this for at the resubmission phase):
      • Section 3 subsections 3.4, 3.5: Inference for multiple regression
      • Section 4: Discussion. You will write this conclusion based on the results of sections 3.4 and 3.5.
  3. The group leader only will submit the following files on Moodle on behalf of the whole group:
    1. The project_submission.Rmd R Markdown file with sections 1, 2, 3.1, 3.2, and 3.3 completed
    2. The project_submission.html HTML report file.
    3. Only if your data is confidential or has private information, submit your Excel or CSV file as well. Otherwise your should publish your data on Google Sheets.

4. Project Resubmission

  1. Re-read the previous sections of the example project resubmission you read earlier. In addition, read the following sections you skipped previously:
    • Section 3 subsections 3.4, 3.5: Inference for multiple regression
    • Section 4: Discussion.
  2. Using the same project_submission.Rmd file you submitted for the project submission phase:
    • Incorporate any feedback given to you from the project submission phase.
    • Complete Sections 3.4 and 3.5: Inference for multiple regression
    • Complete Section 4: Discussion. You will write this conclusion based on the results of sections 3.4 and 3.5.
  3. The group leader only will submit the following files on Moodle on behalf of the whole group:
    1. The updated project_submission.Rmd R Markdown file.
    2. The updated project_submission.html HTML report file.
    3. Only if your data is confidential or has private information, submit your Excel or CSV file as well. Otherwise your should publish your data on Google Sheets.
  4. Everyone must complete the exit survey Google Form (to be posted at the end of semester). There are four sections:
    1. Evaluating your peers’ contributions for the project.
    2. Questions about you.
    3. General questions about the course.

Grading Rubric

  • You are only graded on your project resubmission.
  • Your project is worth 30% of your final grade. The 30% is distributed as follows:
    1. 2.5%: Are all necessary files submitted and did you complete the “SDS/MTH 220 Exit Survey” Google Form?
    2. 27.5%: Based on this rough guideline criteria.
